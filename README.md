# AI Gigaschool — AI-компаньон для профориентации

Диалоговый помощник, который помогает пользователю построить карьерный план: от сбора информации о человеке до рекомендаций профессий и образовательных дорожных карт. Кейс Gigaschool.


Попробовать:
```
[@ai_career_guidance_bot](https://t.me/ai_career_guidance_bot)
```

---

## Задача, аудитория и боль

**Задача:** дать пользователю персонального профориентационного консультанта в формате диалога (Telegram-бот), без необходимости идти к живому консультанту и без разрозненных тестов и статей.

**Для кого:**
- Школьники и студенты — выбор направления и профессии.
- Работающие взрослые — смена карьеры, переквалификация.

**Боль сейчас:**
- Неясно, с чего начать и как связать интересы с профессиями.
- Профориентационные тесты разбросаны, результаты не объединены в единый портрет и план.
- Нет одного места, где можно и поговорить, и получить конкретные профессии, и увидеть пути обучения (курсы, программы).

## Что делает PoC на демо

- **Диалоговые графы:** управление переходами состояний агента (WHO → ABOUT → TEST → RECOMMENDATION → TALK) задаётся графом диалога; переходы между узлами — по явным условиям и результатам tool calls, а не по произвольному выводу LLM. На базе графа реализуется защита от prompt-injection, jailbreak и утечки системного промпта (см. [docs/governance.md](docs/governance.md)).
- **Сбор информации о пользователе:** имя, возраст, статус (школьник / студент / работающий), интересы, цели — через естественный диалог в рамках узлов графа (WHO, ABOUT).
- **Профориентационные тесты:** выбор и проведение теста (в т.ч. заготовленные вопросы), учёт результатов в портрете (узел TEST).
- **Рекомендации профессий:** на основе диалога и тестов выдаётся список профессий с краткими описаниямию
- **Детали по профессии и дорожные карты:** по выбранной профессии — подробное описание и подбор образовательных программ через RAG (профессии из HH-агрегации, курсы Stepik, Поступи, Нетология, Skillfactory и т.д.).

- **Сохранение контекста:** история диалога и метаданные пользователя хранятся (SQLite), текущее состояние сессии восстанавливается при возврате.

## Что НЕ делает PoC (out-of-scope)

- **Трудоустройство:** поиск вакансий, отклики, резюме — не входит в PoC.
- **Официальная психологическая/медицинская диагностика:** тесты носят информационный характер, не заменяют консультацию специалиста.
- **Мультиязычность и глубокая локализация:** фокус на русскоязычном сценарии.
- **Веб-интерфейс и мобильное приложение:** только Telegram-бот.
- **Масштабирование и отказоустойчивость:** PoC рассчитан на демо/ограниченную нагрузку, без полноценного production SLO.
